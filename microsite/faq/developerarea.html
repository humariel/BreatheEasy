Developer Area

Here we explain everything about the implementation of the project and how to run it.

Technologies

The project is currently using the following technologies/devices:
  • Real Sensors
  • LoRa
  • External data API's
  • MQTT
  • MongoDB
  • Node.js
  • JSON Web Token
  • Nuxt and Vue.js

Implementation

There are two sources of data: external API's and sensors. We use http requests to retrieve data from the API's and to send the sensors and API's data to our platform. The sensors communicate with a gateway, and this is the one responsible for sending the sensor's data to our upper layers.

Our broker is built with MQTT. It uses a database in MongoDB to save it's data permanently. The sensors and external API's communicate with the broker through an API. The API's documentation is available here. If you wish to add any data to our system, you can follow that document that details how the requests must be done to interact with our broker, given that you have permission to do so.

The broker communicates with the client side through an entity that handles authentications (supported with JSON Web Tokens). It also has a database in MongoDB to hold users information. The communication with this entity is done through an API which is documented here.

The client interacts with a website built with Nuxt.js and Vue.js. The divisions in the map are taken from geojson files. They represent the devices that give us information.
